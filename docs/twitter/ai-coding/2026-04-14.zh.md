### 1.1 Claude Code 桌面重设计与 Routines 发布（🡕）

Anthropic 同一天向 Claude Code 推出了两项重大更新。官方 [@claudeai](https://x.com/claudeai) 账号 [宣布了桌面重设计](https://x.com/claudeai/status/2044131493966909862)，包含并排运行的并行会话、用于管理会话的新侧边栏、集成终端、文件编辑、HTML/PDF 预览，以及拖放式布局。[@9to5mac](https://x.com/9to5mac) [报道了这次发布](https://x.com/9to5mac/status/2044133564384502179)，并披露了推文中没有提到的第二个功能：Routines——即使用户的 Mac 离线，也会在 Anthropic Web 基础设施上运行的定时自动化。[9to5Mac 文章](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/) 详细说明了按档位划分的限制：Pro 每天 5 个 routine，Max 每天 15 个，Team/Enterprise 每天 25 个。Routines 发布时自带仓库和连接器访问，免去了用户自行管理 cron job 和 MCP server 基础设施的需要。

[@Yuchenj_UW](https://x.com/Yuchenj_UW) [把这次重设计框定为 IDE 范式转变](https://x.com/Yuchenj_UW/status/2044133573326934384)，并引用 Andrej Karpathy 的话：“我们会需要一个更大的 IDE。基本单位不是文件，而是智能体。” 其含义是：“Cursor 现在也必须争夺 IDE 未来的定义权。” [@RoundtableSpace](https://x.com/RoundtableSpace) [放大了这种兴奋感](https://x.com/RoundtableSpace/status/2044185178063679580)，获得当天最高浏览量（4,898 浏览，13 收藏），并引来 [@traleyweb3](https://x.com/traleyweb3) 这样的回复：“claude code 刚刚获得了一次重大升级。”

**讨论洞察：** 这次重设计获得了广泛传播，但没有出现技术性批评。Routines 功能——服务端定时自动化——在结构上更重要，却比视觉重设计获得了更少社交关注。

**与前一天对比：** 昨天的报告追踪了 Copilot CLI 走向远程和移动端，以及 Codex 转变为桌面 Super App。今天，Claude Code 通过让桌面应用支持多会话并增加服务端自动化作出回应——既匹配“设备无关”的方向，又增加了竞争对手尚未宣布的调度能力。

### 1.2 Claude Code Skills 生态扩展（🡕）

两篇帖子展示了可安装 skills 生态的增长，这些 skills 扩展了 Claude Code 开箱即知的内容。

[@shi3z](https://x.com/shi3z) [发布了一个面向 SparkJS 的 Claude Code skill](https://x.com/shi3z/status/2044202378140995643)，SparkJS 是基于 Three.js 构建的 3D Gaussian Splat 渲染库。该帖获得当天最高分（95.8），有 22 个赞、12 次收藏和 3,240 次浏览。[GitHub 仓库](https://github.com/shi3z/sparkjs-skill) 包含一个 SKILL.md 和 references 目录，覆盖完整 SparkJS API：SparkRenderer、SplatMesh、PackedSplats、基于 SDF 的 splat 编辑、Level-of-Detail streaming、带 117+ stdlib functions 的 Dyno shader graph system，以及平台特定性能优化。安装只需要执行一次 `git clone`，目标为 `~/.claude/skills/`。

另外，[@hasantoxr](https://x.com/hasantoxr) [重点介绍了 Taste Skill](https://x.com/hasantoxr/status/2044008552231252248)，这是一个开源 agent skills 集合，能让 AI 编程工具具备“设计品味”。该 [仓库](https://github.com/LeonxLnx/taste-skill) 拥有 8.3k star 和 827 个 fork，包含七种 skill 变体（taste-skill、redesign-skill、soft-skill、brutalist、minimalist、stitch、output-skill），以及三个可调旋钮，分别控制布局变化、动效强度和视觉密度。与 SparkJS skill 不同，Taste Skill 可跨工具使用：Cursor、Claude Code、Codex、Windsurf、Copilot 和 Antigravity。

![Taste Skill GitHub README，显示 8.3k star、skill 变体表格和 CLI 安装命令](https://pbs.twimg.com/media/HF3G8lLbgAA4ErV.png)

**讨论洞察：** SparkJS skill 展示了领域专用知识注入（3D 图形 API），而 Taste Skill 展示了横向质量注入（设计模式）。两者共同展示了两种不同的 skill 模式：深度垂直专业能力 vs. 宽泛横向质量能力。

**与前一天对比：** 昨天的报告没有专门呈现 skills。skills 生态正与 MCP 一起成长，成为 AI 编程智能体的第二种扩展机制。

## 2. 令人困扰的问题

### Claude Code 成本不透明（Medium）

[@GithubAwesome](https://x.com/GithubAwesome) [直接描述了这个问题](https://x.com/GithubAwesome/status/2044203308642968037)：“有人每天在 Claude Code API credits 上花超过 $200，却完全不知道钱花到哪里去了。” 解决方案 codeburn 是一个 Terminal UI，会读取本地会话 transcript，并把每个 token 支出按任务类型（debugging、brainstorming、coding）归类——它把“臃肿 MCP servers 造成的 token 开销”识别为成本驱动因素。该工具包含交互式每日活动图表和 Mac 菜单栏小组件。分析不需要任何 LLM 调用。

这延续了昨天报告中的 token 膨胀挫败感，当时 Claude Code v2.1.100+ 已确认每次请求会注入约 20k 服务端 token，而且系统开销在任何用户消息之前就消耗约 50k token。今天的证据增加了一个新的成本向量：臃肿 MCP servers 也会造成 token 浪费。

### Claude Code 流失信号（Low）

[@inancgumus](https://x.com/inancgumus) [简单表示](https://x.com/inancgumus/status/2044202972578467949)：“我已经彻底摆脱 Claude Code 了。” 该帖获得 374 次浏览，但没有点赞和收藏——粉丝看到了，但没有放大。作者没有解释切换原因。作为孤立数据点，它代表一个没有诊断价值的流失信号，但其背景是多天以来持续记录的成本与质量担忧。

## 3. 人们期望的功能

### 精细化 token 成本归因

“Claude Code API credits 每天 $200”（[@GithubAwesome](https://x.com/GithubAwesome)）与内建成本可见性之间仍存在巨大缺口。Codeburn 读取本地会话 transcript 解决这一点，但它是外部方案，而且只能事后分析。用户需要内建在 agent 中的实时、按任务成本归因——在账单到来之前，而不是之后，就知道一次调试会话的成本可能是一次代码生成会话的 3 倍。昨天的报告也标记了同样需求；今天的数据用一个具体金额（$200/day）确认了这一点。

### 跨工具 Skill 可移植性

Taste Skill 的跨工具兼容性（Cursor、Claude Code、Codex、Windsurf、Copilot、Antigravity）凸显了多数 skills 缺少的东西：可移植性。SparkJS skill 只适用于 Claude Code。随着 skills 生态增长，开发者会想要一次安装、到处使用——类似 MCP 如何标准化工具连接，一个跨 agent 可用的 skill registry 和格式会变得必要。Taste Skill 使用的 `npx skills add` 命令暗示了这个方向。

## 6. 新动态与亮点

### Claude Code Routines：服务端定时自动化

Anthropic 在 research preview 中发布 Routines——在 Claude Code Web 基础设施上运行、可访问仓库和连接器的定时自动化。这让 Claude Code 从反应式工具（用户提示，智能体回应）转向主动式工具（智能体在没有用户在场时按计划运行）。档位限制：Pro 5/day、Max 15/day、Team/Enterprise 25/day。([9to5Mac](https://9to5mac.com/2026/04/14/anthropic-adds-repeatable-routines-feature-to-claude-code-heres-how-it-works/), [推文](https://x.com/9to5mac/status/2044133564384502179))

### Claude Code 桌面端支持多会话

重新设计后的 Claude Code 桌面应用支持在单个窗口中并行运行多个会话，并配有管理侧边栏。新增集成终端、文件编辑、HTML/PDF 预览，以及拖放式布局中的更快 diff viewer。这直接回应了开发者在代码库不同部分并行运行智能体任务的工作流。([推文](https://x.com/Yuchenj_UW/status/2044133573326934384))

### SparkJS Skill：首个面向 3D 图形的领域专用 Claude Code Skill

一个 MIT 许可的 skill，让 Claude Code 完整了解 SparkJS 3D Gaussian Splat 渲染库——覆盖 renderer、splat mesh、SDF editing、LoD streaming 和 117+ shader stdlib functions。通过单次 git clone 安装。它展示了把领域专用 API 知识打包成可移植 skill 文件的模式。([GitHub](https://github.com/shi3z/sparkjs-skill), [推文](https://x.com/shi3z/status/2044202378140995643))

## 7. 机会在哪里

**[+++] 智能体 Skill Registry 与跨工具可移植性** — SparkJS Skill（仅 Claude Code）和 Taste Skill（6+ 工具）显示两种 skill 分发模式正在同时出现。Taste Skill 的 `npx skills add` 和跨工具支持拥有 8.3k star；SparkJS 的 Claude-only git clone 方案在数小时内获得 22 个赞和 12 次收藏。一个统一的 skill registry——skill 只需发布一次，就能安装到 Claude Code、Cursor、Codex、Windsurf 和 Copilot——将捕获 MCP 在工具连接中捕获的同类生态价值。需求存在；标准尚不存在。

**[++] 面向 AI 编程智能体的实时成本归因** — 昨天的报告记录了用户消息前约 50k token 的系统开销。今天的数据增加了 $200/day 的成本数字，以及 codeburn 这个首个外部方案。机会在于把成本归因内建到 agent 本身：执行前的按任务成本估算、预算提醒和每结果成本指标。每个 Claude Code API 用户都是这项能力的潜在客户。

**[++] Anthropic 之外的定时智能体自动化** — Claude Code Routines 是首个面向 AI 编程智能体的服务端定时自动化。该功能受档位限制（5-25 次/天），且仅限 Anthropic。一个开源等价物——可使用任意 LLM 后端、运行在开发者自有基础设施上的定时智能体任务——将服务与昨天推动 Multica 达到 4,000+ star 的同一需求（自托管智能体基础设施）。

**[+] AI 辅助教育平台** — Ship Your First App 展示了 Claude Code 作为面向非技术用户的结构化辅导平台。课程模式（获取指令、引导构建、互动练习）可以从 Web 开发泛化到任何技术领域。缺口在于 Phase 1 是唯一可用内容——Modules 1-3 和 Phases 2-3 全都标记为“Coming Soon”。最早在多个领域提供完整课程的玩家将拥有这一类别。
