### 1.1 Ollama 推出 Copilot CLI 集成，解锁免费的本地智能体运行 🡕

当天最主要的事件是 [@ollama](https://x.com/ollama) [宣布原生支持 GitHub 的 Copilot CLI](https://x.com/ollama/status/2045686038274990147)（407 点赞，113 收藏，36,321 浏览量）。该集成让用户可以通过 Ollama 托管模型，在本地硬件上完整运行 Copilot CLI 终端智能体：探索议题和拉取请求、从工单搭起工作骨架、浏览陌生代码库。命令是 `ollama launch copilot`。在回复串中，Ollama [感谢 @scaryrawr](https://x.com/ollama/status/2045686038274990147) 的贡献，并向一名觉得 Pro 方案用量扣得太少的用户提供缓存命中排查建议。

[@itsafiz](https://x.com/itsafiz) [发布了一份分步指南](https://x.com/itsafiz/status/2045908305688629348)（22 点赞，24 收藏，2,002 浏览量），介绍如何零成本用 Ollama 运行 Copilot CLI，并用 Haystack 构建完整 RAG 流水线。

![GitHub Copilot CLI v1.0.32 使用本地 Ollama 模型构建 Haystack RAG 流水线，显示文件编辑和目录探索](https://pbs.twimg.com/media/HGSGvy5WgAAvEr_.jpg)

截图显示 Copilot CLI v1.0.32 正在配合本地 Qwen 模型运行，探索目录、创建 requirements.txt，并搭起 PDFRAGPipeline 类。[@BradGroux](https://x.com/BradGroux) [放大了这条公告](https://x.com/BradGroux/status/2045701436328075413)（31 点赞，10 收藏，6,308 浏览量），[@asamassekou10](https://x.com/asamassekou10) [指出](https://x.com/asamassekou10/status/2045734578879373686)，Copilot CLI 原生支持在本地探索议题和拉取请求是“彻底改变游戏规则”，而他已经在另一个项目中运行 Gemma 4。

**与前日对比：** 昨天 Copilot CLI 的报道聚焦自动模型选择正式可用，以及 10% 的高级用量折扣。今天故事转向通过 Ollama 本地执行，它完全消除 API 成本，并把 Copilot CLI 定位为第一个通过官方集成提供全本地选项的主要智能体。

![Anthropic Fin AI Agent 支持回复确认后端同步问题，但无法修复、升级给真人或提供补偿](https://pbs.twimg.com/media/HGP4QgebwAAQrJH.png)

## 7. 机会在哪里

**[+++] 本地优先智能体基础设施** — Ollama 的 Copilot CLI 集成（1,159 分，36K 浏览量）证明，终端智能体的本地模型服务是可行且需求很高的。M5 Max 上的 Qwen 3.6 35B 能以约 90tps 提供无瑕编码表现（[@ivanfioravanti](https://x.com/ivanfioravanti)，[帖子](https://x.com/ivanfioravanti/status/2045853328819515540)）。缺口在于：当前没有工具能自动做本地模型选择、本地资源不足时回退到云端，或比较本地与云端执行路径的成本。提供商限流、订阅不透明和已验证的本地模型质量结合起来，让这是一个结构性且持续增长的需求。

**[+++] 跨智能体审查与验证工作流** — [@drivelinekyle](https://x.com/drivelinekyle) 展示了在 Claude Code 会话内用 Codex 审查交错执行的 code-review-fix-next 协议（[帖子](https://x.com/drivelinekyle/status/2045873804841271625)）。[@Viewforge](https://x.com/Viewforge) 表明即便 Codex 也会在长时间运行的 computer-use 任务中失败（[帖子](https://x.com/Viewforge/status/2045924743342334263)）。一个自动化跨智能体验证层——让一个智能体在执行过程中持续审查另一个智能体的输出——可以同时回应昨天的质量关卡需求（#4 opportunity）和多工具编排需求。手动交错模式已经存在；机会在于产品化。

**[++] AI 编程订阅情报** — [@GraemeVIP](https://x.com/GraemeVIP) 展示了 20 GBP Claude 订阅相当于 $103 API 等价用量（[帖子](https://x.com/GraemeVIP/status/2045674250460144107)）。[@Jasonzhouzld](https://x.com/Jasonzhouzld) 记录了 Anthropic 支持团队承认后端 bug 却无法修复（[帖子](https://x.com/Jasonzhouzld/status/2045751646542819657)）。[@johncrickett](https://x.com/johncrickett) 指出企业会为了人工支持渠道选择 Copilot（[帖子](https://x.com/johncrickett/status/2045760498352722181)）。一个订阅健康监控可以跨提供商追踪权益状态、提醒同步问题、比较实际使用与订阅价值，并推荐最优提供商分配，同时服务个人开发者和企业买家。

**[++] 面向 AI 编程的设计系统集成** — awesome-claude-design 为 Claude Code 提供 68 个设计系统（[帖子](https://x.com/sharbel/status/2045809668660850703)），解决 “通用灰色 UI” 问题，但仅限 Claude。把设计系统上下文扩展到 Codex、Antigravity、Cursor 等工具，并建立社区贡献的设计系统市场，可以捕获更广泛的非通用 AI 生成 UI 需求。

**[+] 非编程智能体技能市场** — Minimalist Entrepreneur skills（[帖子](https://x.com/chenzeling4/status/2045813354854240306)）展示了 Claude Code 技能系统用于商业战略，而不是代码。[@VaibhavSisinty](https://x.com/VaibhavSisinty) [列出](https://x.com/VaibhavSisinty/status/2045853757062398070) Codex 能力，覆盖文件整理、CSV 合并、报告生成、网页浏览和应用控制。一个面向产品经理、分析师、营销人员和创始人的非编程智能体技能精选市场，可以把编程智能体平台的可服务市场扩展到开发者之外。

## 8. 要点总结

1. **Ollama 的 Copilot CLI 集成是当天最关键事件。** 这条推文得分 1,159、浏览量 36,321，是当天最高互动内容，领先 5x。运行 `ollama launch copilot` 后，任何人都可以在本地模型上以零 API 成本使用 GitHub 终端智能体。这是主要本地模型服务器与主要云原生编程智能体之间的第一个官方桥接。（[@ollama](https://x.com/ollama)，[帖子](https://x.com/ollama/status/2045686038274990147)）

2. **Codex computer-use 已经在真实基础设施任务中有记录，也在竞争中暴露失败。** 一名用户用 Codex 自主导航 Tailscale 管理后台，把 Raspberry Pi 配成 VPN 出口节点（[@_julianschiavo](https://x.com/_julianschiavo)，[帖子](https://x.com/_julianschiavo/status/2045757690081919312)）。另一个 Viewforge 基准测试显示，Codex + GPT-5.4 没能跑通一个第三方 computer-use 套件成功跑通的 YouTube-to-TikTok 任务（[@Viewforge](https://x.com/Viewforge)，[帖子](https://x.com/Viewforge/status/2045924743342334263)）。Computer-use 可用于明确任务，但会在长时间运行的多步工作流中崩掉。

3. **供应商信任侵蚀因支持系统失灵的证据而加深。** Anthropic 的 AI 支持智能体确认后端订阅同步 bug，却明确表示无法修复、无法升级给人工，也无法补偿（[@Jasonzhouzld](https://x.com/Jasonzhouzld)，[帖子](https://x.com/Jasonzhouzld/status/2045751646542819657)）。一个 60+ 账号的企业组织被封禁，申诉路径只有 Google Form（[@johncrickett](https://x.com/johncrickett)，[帖子](https://x.com/johncrickett/status/2045760498352722181)）。昨天这个主题还停留在不透明政策，今天已经进入支持系统失灵阶段。

4. **多工具工作流已经有具体交错协议。** 实践者不只是说“两个都用”，而是在发布具体模式：在 Claude Code 中每隔一步用 Codex 审查，防止漂移积累（[@drivelinekyle](https://x.com/drivelinekyle)，[帖子](https://x.com/drivelinekyle/status/2045873804841271625)）。Google 自家员工也背书把 Claude Code、Codex 与 Google 工具一起用（[@LyalinDotCom](https://x.com/LyalinDotCom)，[帖子](https://x.com/LyalinDotCom/status/2045674717487509776)）。多工具模式正在从采用走向方法论。

5. **“vibe coding 后遗症” 命名了新兴反弹。** vibe coding 一边扩散到生活方式文化和初学者教育，一边也让实践者开始命名后果：没有清晰架构的 AI 生成代码库会产生复利式技术债（[@DanTheronX](https://x.com/DanTheronX)，[帖子](https://x.com/DanTheronX/status/2045925420957147462)）。代码质量崩塌和安全事件正在被——即使是讽刺地——连接到 AI 编程实践（[@0xfeMMANUEL](https://x.com/0xfeMMANUEL)，[帖子](https://x.com/0xfeMMANUEL/status/2045978573576544401)）。

6. **本地模型质量获得多份独立报告确认。** M5 Max 上通过 mlx_lm 运行的 Qwen 3.6 35B 提供“非常愉快的编码体验”，速度约 90tps（[@ivanfioravanti](https://x.com/ivanfioravanti)，[帖子](https://x.com/ivanfioravanti/status/2045853328819515540)）。同一模型在 OpenCode 中表现“好得惊人”（[@loktar00](https://x.com/loktar00)，[帖子](https://x.com/loktar00/status/2045974956337512521)）。但运行框架敏感性仍是真问题：在 Hermes 中会“惊艳和彻底灾难之间来回跳”（[@paulmarin90](https://x.com/paulmarin90)）。模型能用；质量变化主要出在集成面。
