### 1.1 Claude Code 遇上 Google Antigravity：集成热潮与反弹（🡕）

这股热情并非没有受到挑战。[@pcshipp](https://x.com/pcshipp) [回击](https://x.com/pcshipp/status/2043643091127824826) 并发布了当天回复数最多的帖子（54 条回复）：“Antigravity 是 Google 创造的最失败产品之一。即使免费，它仍然无法与 Cursor、Codex 和 Claude code 等工具竞争。质量 >> 免费。” 与此同时，[@lMDU1zXxSEbWc5c](https://x.com/lMDU1zXxSEbWc5c) [报告](https://x.com/lMDU1zXxSEbWc5c/status/2043644008917000383) 称“google antigravity 挂了吗？没有响应！”——这个可靠性问题进一步强化了质量抱怨。

### 1.3 GitHub Copilot CLI 走向远程与移动端（🡕）

两篇帖子记录了 GitHub Copilot CLI 能力的一次重要扩展。[@msdev](https://x.com/msdev) [演示](https://x.com/msdev/status/2043769670688461051) 了一个板球计分应用：使用三条 Copilot CLI 命令，在 30 分钟内做出成品，分别是 `/plan` 用来确定范围，`/autopilot` 用来构建，`/fleet` 用来扩展。该帖获得 77 个赞和 47 次收藏。

另外，[@GHchangelog](https://x.com/GHchangelog) [宣布](https://x.com/GHchangelog/status/2043705785843171838) 远程控制 CLI 会话进入公开预览。[更新日志文章](https://github.blog/changelog/2026-04-13-remote-control-cli-sessions-on-web-and-mobile-in-public-preview) 详细说明，`copilot --remote` 会把 CLI 活动实时串流到 GitHub。用户可以从 Web 或移动端监控和引导会话，包括：发送会话中的引导消息、审查和修改计划、在 plan/interactive/autopilot 模式之间切换、批准或拒绝权限请求，以及回应 `ask_user` 提示。`/keep-alive` 命令可防止机器在长任务期间休眠。该功能已在 Android beta 和 iOS TestFlight 上可用。

[@h34j3j8j](https://x.com/h34j3j8j) [询问了差异](https://x.com/h34j3j8j/status/2043774154013389201)：Copilot 在 IntelliJ 中的 “Ask” 模式和 “Agent” 模式有什么不同，并这样描述差别：“Ask 会告诉我在哪里做、做什么。Agent 会真的去做。” 配图显示 IDE 状态栏中的模式下拉菜单（Ask、Edit (Deprecated)、Agent、Plan）。

![GitHub Copilot 模式选择器，显示 Ask、Edit Deprecated、Agent 和 Plan 选项](https://pbs.twimg.com/media/HFzxUyAXQAAqWdc.png)

远程会话、移动访问和 /fleet 扩展命令的汇合，表明 Copilot CLI 正在从本地编程助手转向设备无关的开发平台。

### 1.4 MCP 扩展到企业与 Web3（🡕）

Model Context Protocol 今天出现在三个不同语境中，显示出采用范围正在变宽。

[@rezadorrani](https://x.com/rezadorrani) [演示](https://x.com/rezadorrani/status/2043660480783192338) 了 Power Apps Authoring MCP Server，它可以使用 GitHub Copilot CLI 和 Claude Code 构建 Canvas Apps。在回复中，[@PromptSlinger](https://x.com/PromptSlinger) 提出了关键架构问题：“MCP server 是真的直接操作 canvas app 组件吗？还是更像生成 YAML 再导入？” [@oscar_ster3808](https://x.com/oscar_ster3808) 给出了更宽泛的评估：“MCP 正在悄悄吃掉中间件栈。当 Claude Code 成为企业工具的主界面时，谁能构建什么的问题会彻底改变。”

[@dungeonroll](https://x.com/dungeonroll) [宣布](https://x.com/dungeonroll/status/2043823101377470643) 在 Anthropic MCP Registry 上发布，使使用 Claude Code 或 Claude Desktop 的 AI agents 能够“注册斗士、押注 ETH，并自主战斗至死”。这代表 MCP 使用场景正在从开发者工具扩展到区块链游戏——智能体直接与智能合约互动。

[@DataChaz](https://x.com/DataChaz) [重点介绍了 Multica](https://x.com/DataChaz/status/2043634352933028277)，这是 Claude Managed Agents 的开源克隆，几天内达到 4,000+ GitHub star。Multica 与 Claude Code、OpenAI Codex、OpenClaw 和 OpenCode 集成，提供自托管智能体基础设施，具备可复用 skills、多工作区隔离和共享人类-AI 看板。其工作流——启动 daemon、分配 ticket、隔离 workspace、实时 WebSocket 更新——复刻了 Anthropic 的托管方案，但运行在开发者自己的服务器上。

## 2. 令人困扰的问题

### Claude Code token 膨胀与幽灵 token（High）

三个独立来源记录了 Claude Code 中过度 token 消耗的问题。[@de1lymoon](https://x.com/de1lymoon) 以在 Google 11 年的经验 [报告](https://x.com/de1lymoon/status/2043668578570744085)：“Claude 有 40% 的时间会破坏约定……一个隐藏 bug 烧掉 20,000 个幽灵 token。” 单个 markdown 规则文件把约定违规率从 40% 降到了 3%。[@grok](https://x.com/grok) [佐证](https://x.com/grok/status/2043672440966967661) 称，“较新的 Claude Code 版本（2.1.100+）每次请求会注入约 20k 额外服务端 token，这已由 HTTP 代理测试、Reddit 分析和公开 GitHub issue 确认。” [@a_lamparelli](https://x.com/a_lamparelli) [展示截图](https://x.com/a_lamparelli/status/2043710164964983197)，显示一次没有工具调用的 “continue” 命令就消耗了 2% 会话用量。`/context` 拆解揭示了开销：system prompt 7k tokens（3.5%）、system tools 8.9k（4.5%）和 autocompact buffer 33k（16.5%）——这还没算任何用户消息。

![Claude Code 上下文用量显示 200k token 中已使用 49.5k，并按类别给出详细拆解](https://pbs.twimg.com/media/HFy2jBzWkAANn3I.jpg)

这延续了昨天报告中的缓存 TTL 和 token 成本担忧，今天的证据则增加了具体版本号和独立验证方法。

### Gemini CLI 账号封禁（Medium）

[@championswimmer](https://x.com/championswimmer) [报告](https://x.com/championswimmer/status/2043645123196604478) 称，自己在几乎不使用 Gemini CLI 的情况下遭遇账号封禁，提示信息为“此账号中的该服务因违反服务条款已被停用”。用户指出这属于一种模式：“奇怪的 CLI 工具封禁还在继续。” [@justcardz](https://x.com/justcardz) 建议使用 Google 表单申诉，并称一小时内可以解决。

![Gemini CLI 终端显示账号因违反服务条款被禁用，并给出申诉说明](https://pbs.twimg.com/media/HFx8OiwXkAADi5N.jpg)

### Google Antigravity 质量与可靠性（Medium）

[@pcshipp](https://x.com/pcshipp) 通过 [宣称](https://x.com/pcshipp/status/2043643091127824826)“Antigravity 是 Google 创造的最失败产品之一。即使免费，它仍然无法与 Cursor、Codex 和 Claude code 等工具竞争”引来 54 条回复——是数据集中最高。另有 [@lMDU1zXxSEbWc5c](https://x.com/lMDU1zXxSEbWc5c) [报告](https://x.com/lMDU1zXxSEbWc5c/status/2043644008917000383) 直接宕机。54 条回复的讨论量说明，这不是孤立抱怨，而是广泛存在的挫败感。

### Claude Code 学习曲线（Low）

[@zodchiii](https://x.com/zodchiii) [幽默记录](https://x.com/zodchiii/status/2043709527938207755) 了典型 Claude Code 用户旅程：“输入一个很长的 prompt，得到结果，自己重写一切，发现 Claude 有 /commands，原来 checkpoint 一直都在文档里。” 这个笑话有效，是因为它描述了一种常见经历：用户在学会 /commands、/clear 和斜杠命令等内置功能之前，会浪费 token 和精力。

## 3. 人们期望的功能

### Token 成本透明度与可观测性

[@a_lamparelli](https://x.com/a_lamparelli) 的 [截图](https://x.com/a_lamparelli/status/2043710164964983197) 显示，Claude Code 的 `/context` 命令会给出 token 拆解，但一次 “continue” 带来的 2% 会话消耗对用户仍然无法解释。[@grok](https://x.com/grok) [指出](https://x.com/grok/status/2043672440966967661)，约 20k 服务端 token 注入并未记录在文档中。再加上昨天的缓存 TTL 回归报告，用户持续要求实时成本可见性和可预测计费。这些信息存在于服务端，但没有以可执行形式呈现给开发者。

### 感知约定的 AI 编程智能体

[@de1lymoon](https://x.com/de1lymoon) [展示](https://x.com/de1lymoon/status/2043668578570744085)，单个 markdown 文件就能把 Claude Code 约定违规率从 40% 降到 3%。“开箱可用”和“正确遵守团队约定”之间的差距，目前靠手动配置来填补。一个能自动检测并适应项目约定的智能体——lint 规则、命名模式、架构决策——如果无需手写规则文件，就能解决 Claude Code 最常见的失败模式。

### 移动优先的智能体式开发

GitHub 的 [远程 CLI 会话](https://github.blog/changelog/2026-04-13-remote-control-cli-sessions-on-web-and-mobile-in-public-preview) 开始满足这一需求，但该功能仍处于公开预览阶段，移动应用可用性有限（Android beta、iOS TestFlight）。来自 [@msdev](https://x.com/msdev) 获得 47 次收藏的帖子以及远程会话公告的需求信号表明，开发者希望从任意设备监控、引导和批准智能体工作——而不只是从运行 CLI 的那台机器。

## 6. 新动态与亮点

**GitHub Copilot CLI 远程会话进入公开预览。** `copilot --remote` 命令会把 CLI 活动实时串流到 GitHub，使用户能从 Web 或移动端监控和引导。功能包括会话中消息、计划审查、模式切换，以及从任意设备批准权限。Android beta 和 iOS TestFlight 已可用。([更新日志](https://github.blog/changelog/2026-04-13-remote-control-cli-sessions-on-web-and-mobile-in-public-preview))

**OpenAI Codex “Super App”截图浮出水面。** 首批公开图片显示，Codex 正从聊天界面转变为完整桌面开发环境，带集成浏览器、项目侧边栏和 Basic/Advanced UI 模式。GPT-5.4 Extra High 是默认模型。同一串推中也标记了供应链安全事件（Axios compromise、macOS 签名证书轮换）。([推文](https://x.com/chetaslua/status/2043634180979085785))

## 7. 机会在哪里

**[+++] 强：自托管智能体基础设施**

Multica 几天内获得 4,000+ star，量化了市场对云锁定托管智能体平台的自托管替代方案需求。Anthropic 的 Managed Agents 要求使用其云和定价；企业团队需要能在自有服务器上运行、同时可选择模型（Claude、GPT、开源）的智能体编排。这个模式——daemon、ticket 分配、隔离 workspace、实时更新、可复用 skills——已经被 Multica 证明可行。机会在于把它加固到企业可用：SOC 2 合规、审计日志、基于角色的访问控制，以及与现有 CI/CD pipeline 集成。任何构建智能体基础设施的团队都应该密切关注这一领域。

**[++] 中等：面向 AI 编程工具的 token 成本可观测性**

今天的数据为昨天的缓存 TTL 担忧增加了具体证据。[@a_lamparelli](https://x.com/a_lamparelli) 的 `/context` 截图显示，系统开销（prompt + tools + autocompact buffer）在单条用户消息之前就消耗约 50k token。[@grok](https://x.com/grok) 引用了 v2.1.100+ 中每次请求约 20k 额外服务端 token。[@de1lymoon](https://x.com/de1lymoon) 报告每个隐藏 bug 会消耗 20,000 个幽灵 token。一个能让这些成本可见、可预测、可优化的代理或仪表盘，将服务每个触达限制的 Claude Code 用户。数据存在，但没有以可操作方式呈现出来。

**[++] 中等：MCP 作为通用中间件层**

MCP 今天出现在三个彼此无关的语境中：企业低代码（Power Apps）、Web3 游戏（Dungeon Roll）和智能体基础设施（Multica）。[@oscar_ster3808](https://x.com/oscar_ster3808) 捕捉到了趋势：“MCP 正在悄悄吃掉中间件栈。” 机会在 MCP 工具链——发现、测试、安全扫描和 MCP server 监控——因为该协议正在成为 AI 智能体与外部系统之间的默认集成层。Anthropic MCP Registry 是正在出现的应用商店；帮助开发者发布、验证和维护 MCP server 的工具可以填补基础设施缺口。

**[+] 新兴：非编码者 AI 熟练度工具**

[@clairevo](https://x.com/clairevo) 对 Cowork 的描述——“教你 Claude Code 心智模型，而不需要打开终端”——以及 [@degensing](https://x.com/degensing) 对九类非开发工程角色 AI 工具要求的梳理，都指向一个增长中的受众：他们需要 AI 熟练度，但不具备 CLI 熟练度。“brain” 文件模式（一个告诉 AI 你是谁、你如何工作的持久上下文文档）是一种工作流创新，可从 Cowork 泛化到任何 AI 工具。面向特定角色的 AI 熟练度模板和培训材料（QA、DevOps、SRE、Data）代表一个服务不足的市场。

**[+] 新兴：设备无关的智能体式开发**

GitHub 的远程 CLI 会话和 Codex 的桌面 Super App 都在解决同一个限制：智能体式编程工作流被锁定在单一设备上。机会在于能支持设备间无缝交接的工具层——在工作站上启动任务，从手机监控，从平板批准权限。GitHub 通过 `copilot --remote` 拥有先发优势，但这种模式可泛化到任何 CLI 智能体。面向移动端的智能体引导界面（不只是监控）是下一步。
